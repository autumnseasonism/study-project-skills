# 提示词文档化模板

第三阶段使用的翻译文档和索引模板。

## 翻译文档

文件命名规则：`ai_analysis/translated_prompts/[原文件路径]_[功能描述]_prompt_zh.md`（路径中的 `/` 用 `_` 替换，并去掉文件扩展名，如 `src/agents/chat.py` -> `src_agents_chat_对话系统_prompt_zh.md`）

文档结构：

```markdown
# 提示词翻译文档

## 元信息
- 原文件位置: `path/to/original/file.py:line_number`
- 变量名称: `variable_name`
- 功能模块: [描述该提示词所属的功能模块]
- 调用场景: [描述何时/如何触发此提示词]
- 来源分类: [生产代码 / 测试代码]

## 英文原文
[保留完整的英文原文，包括所有格式和换行]

## 中文翻译
[中文翻译内容]

## 关键参数
- [列出提示词中使用的变量/占位符，说明每个参数的含义和来源]

## Few-shot 示例
[如果提示词中包含示例，单独列出；没有则写"无"]

## 提示词链路
- 上游: [该提示词的输入来自哪里——哪个函数调用、哪个用户输入]
- 下游: [该提示词的输出传递给谁——返回给用户、传给下一个提示词、存入数据库]

## 相关代码上下文
[简要说明该提示词在代码中的使用方式，包括所在函数名、调用条件等]
```

## 翻译质量检查清单

在写每份翻译文档时，确认以下几点：
- 占位符 `{variable}`、`{{template}}`、`${expr}` 保持原样不翻译
- JSON/YAML/XML 等结构标签保持原样
- 技术术语准确（"chain of thought" -> "思维链"，"few-shot" -> "少样本"，"embedding" -> "向量嵌入"）
- 如果原始提示词已经是中文，只标注位置，跳过翻译，在文档中注明"原文为中文，无需翻译"

## 索引文件

维护 `ai_analysis/translated_prompts/INDEX.md`，格式如下：

```markdown
# 提示词索引

## 按功能模块分组

### [模块名称 1]
| 提示词名称 | 原文件位置 | 类型 | 翻译文档 |
|-----------|-----------|------|---------|
| system_prompt | `src/agent.py:15` | 系统提示词 | [链接](./xxx_prompt_zh.md) |

### [模块名称 2]
...

## 统计
- 总计: X 个提示词
- 生产代码: X 个
- 测试代码: X 个
- 中文提示词（跳过翻译）: X 个
```
